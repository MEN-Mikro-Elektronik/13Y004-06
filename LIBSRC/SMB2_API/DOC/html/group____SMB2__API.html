Install alert callback function.

The alert callback function will be invoked if the specified SMBus device reports an alert.

The function requires a free UOS_SIG_USRx signal number and returns SMB_ERR_ALERT_NOSIG if no one is available. In this case, the SMB2API_AlertCbInstallSig function should be used to specify an individual UOS library conform signal code.

Issue a read byte command to the Alert Response Address.

If an alarm is returned by a device and the device address matches the address passed, alertCntP returns with an 1 set.

Furthermore the alert callback function (if any installed) will be called for any device address that returns an alert.

char* __MAPILIB SMB2API_Errstring (  int32  errCode,
char *  strBuf
) 
 

Convert SMB2 and MDIS error code to string.

SMB2API_Errstring() creates an error message for error errCode and returns a pointer to the generated string with the following format:

 * ERROR <errtype> <errcode>: <errdescr>
 *

where errtype describes if the error code comes from the system or the driver, errcode describes the error code in hexadecimal format or in the operating system’s native format. errdescr is the corresponding error message string.

Examples: 

    ERROR (MDIS) 0xE00E0f05: SMB2: Bus collision
    ERROR (MDIS) 0x1103: MK: illegal parameter
    ERROR (OS9) #000:221: module not found

Parameters:
errCode IN error code from SMB2 function
strBuf OUT filled with error message (should have space for 512 characters, including '\0')
Returns:
strBuf
Examples:
smb2_boardident.c, smb2_ctrl.c, smb2_f601.c, smb2_simp.c, and smb2api.c.
